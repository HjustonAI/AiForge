Czym jest FORGE?

FORGE to osobisty meta-system w Claude Cowork. Baza operacyjna power usera, z której tworzysz skille, narzedzia, subagentów i prompty do dowolnego AI. Jednoczesnie laboratorium badawcze do odkrywania technik context engineeringu i granic Claude Cowork.

Kluczowa innowacja: Context Inheritance — system dziedziczenia kontekstów inspirowany programowaniem obiektowym, ale dzialajacy na plikach tekstowych. Konteksty kaskaduja jak style CSS — od ogólnych do szczególowych.

Context Inheritance

Konteksty dziedzicza jak klasy w OOP. Kazdy _.ctx.md to "index" folderu — automatycznie dziedziczy po rodzicu. Konteksty kaskaduja od ogólnych do szczególnych, jak style CSS.

Przyklad kompilacji kontekstu

User: "Stworz prompt do Veo3 — kot w kosmosie"

1. Context Engine starts resolution...
2. Read: contexts/_.ctx.md ← bazowe zasady tworzenia
3. Read: contexts/creative/_.ctx.md ← rozszerzenie kreatywne
4. Read: contexts/creative/visual/_.ctx.md ← specyfika wizualna
5. Read: contexts/creative/visual/video-gen.ctx.md ← detale wideo + Veo3

Compiled context = merge(root + creative + visual + video-gen)
→ Prompt-Smith receives full context chain + user intent

Smith Modules — Fabryka Artefaktow

Kazdy Smith to wyspecjalizowany modul tworzenia. Wszystkie korzystaja z Context Engine i zapisuja wyniki do Arsenal.

💬

Prompt-Smith

Opisujesz po swojemu co chcesz osiagnac. Prompt-Smith kompiluje odpowiedni kontekst, analizuje intencje, i generuje zoptymalizowany prompt pod docelowe AI.

"Chce filmik z kotem w kosmosie"
→ [kontekst: creative/visual/video-gen]
→ cinematic prompt optimized for Veo3

najczesciej uzywany multi-target
🛠

Skill-Smith

Tworzy kompletne skille Cowork — SKILL.md, skrypty, referencje. Integruje sie z systemem ewaluacji skill-creator.

"Skill do analizy PDF-ow badawczych"
→ SKILL.md + scripts/ + references/
→ registered in .claude/skills/

cowork-native eval-ready
🔧

Tool-Smith

Tworzy standalone narzedzia — skrypty Python, bash, utility. Testuje w sandboxie przed zapisem do arsenalu.

"Skrypt do batch rename plikow"
→ Python script + docs
→ tested in sandbox

scripts utilities
🤖

Agent-Smith

Tworzy definicje subagentow — wyspecjalizowanych asystentow z wlasnym kontekstem, narzedziam i ograniczeniami.

"Agent do review promptow"
→ agent definition + context
→ ready to spawn

subagents delegation

The Forge Loop

Kazda praca w FORGE podlega cyklowi ciagiego doskonalenia.

💡 THINK

Zdefiniuj co chcesz osiagnac. Opisz naturalna mysl.

▼

🔥 FORGE

Stwórz artefakt: prompt, skill, tool, agent. Smith + Context Engine.

▼

🔬 TEST

Przetestuj w sandboxie. Sprawdz czy dziala jak zamierzono.

▼

🔍 REFINE

Ulepsz na podstawie wynikow. Iteruj az bedzie idealne.

▼

🚀 DEPLOY

Przenies do arsenalu lub bezposrednio do projektu docelowego.

Zasady FORGE

01
Context is Code — Pliki kontekstu to program. Traktuj je z taka sama dyscyplina jak kod zrodlowy. Kazda zmiana w kontekscie zmienia zachowanie systemu.
02
Progressive Disclosure — Laduj tylko to co potrzebne. Orchestrator → modul → szczegoly. Nie zasmiecaj kontekstu informacjami, ktore nie sa teraz potrzebne.
03
Inheritance over Repetition — Nigdy nie powtarzaj kontekstu. Dziedzicz i rozszerzaj. DRY principle przeniesiony na natural language.
04
Arsenal First — Kazdy stworzony artefakt trafia do arsenalu. Reuzywalnosc ponad jednorazowosc. Buduj biblioteke, nie jednorazowki.
05
Experiment Everything — Kazda nowa technika to potencjalny eksperyment. Dokumentuj odkrycia. Dziennik jest rownowazny z kodem.
06
Teach by Building — Sam projekt jest najlepszym tutorialem Claude Cowork. Budujac FORGE, uczysz sie systemu. Meta-learning w akcji.
